## Story

**As a** visitante,
**I want** ver os 3 planos de protocolo lado a lado com toggle Anual/Mensal,
**so that** eu compare preços antes de assinar.

## Acceptance Criteria

1. `/assinatura` exibe os 3 protocolos lado a lado com itens inclusos (checkmarks).
2. Toggle Anual/Mensal recalcula e exibe o preço correto em cada card ao vivo.
3. "Ritual de Autoridade" tem destaque visual (borda + badge "Mais Popular").
4. Nenhuma ação de assinatura funcional ainda (ação fica para o Epic 2) — botão visível porém desabilitado/placeholder.
5. Responsivo mobile-first.

## Dev Notes

### Previous Story Insights

`PricingToggle.tsx` já tem exemplo de código completo na Architecture (Seção 8.1) — usar literalmente esse padrão, não reinventar a interação de tabs. `getProtocols()` (Story 1.2) já ordena por `monthly_price_cents DESC`, mas a ordem de exibição dos 3 cards na página é a ordem de negócio (Cuidados Diários, Ritual de Autoridade, Implante & Alopecia — brief original), não necessariamente a ordem de preço — confirmar com o design se a ordem de exibição precisa de campo próprio ou se a ordem do array retornado já serve. [Source: docs/stories/1.2.story.md]

### State Management (crítico — não duplicar dado de servidor)

[Source: architecture.md Seção 8.2]

```typescript
interface BillingToggleState {
  interval: 'monthly' | 'annual';
  setInterval: (interval: 'monthly' | 'annual') => void;
}
```

Zustand aqui é só o índice de qual preço mostrar — o preço em si sempre vem de `protocols.monthly_price_cents`/`annual_price_cents` já carregados do servidor, nunca recalculado/duplicado no client.

### Data Model

[Source: architecture.md Seção 4.2]

```typescript
interface Protocol {
  id: string; slug: string; name: string;
  productIds: string[]; // via protocol_products
  monthlyPriceCents: number;
  annualPriceCents: number;
  oneTimeEquivalentPriceCents: number;
  isFeatured: boolean;
  stripePriceIdMonthly: string;
  stripePriceIdAnnual: string;
}
```

### Accessibility (WCAG AA)

[Source: architecture.md Seção 8.5]

- `PricingToggle`: `role="tablist"`/`role="tab"`/`aria-selected` (já é o exemplo de referência do documento).
- Nenhum `<div onClick>` nos cards ou botões.

### Technical Constraints

- **CTA de assinatura é placeholder** — não implementar `/api/checkout/subscription` nesta story (Story 2.4).
- **Preço nunca calculado no client** — sempre lido do dado já carregado do servidor via toggle, nunca uma fórmula de desconto reimplementada em JS.

### Testing

[Source: architecture.md Seção 14]

- `PricingToggle.test.tsx`: alterna preço exibido ao clicar em Anual (exemplo já existe na Architecture Seção 14.3 — reaproveitar como base), asserção `jest-axe`.
- `ProtocolCard.test.tsx`: badge "Mais Popular" só aparece no protocolo com `isFeatured: true`.
