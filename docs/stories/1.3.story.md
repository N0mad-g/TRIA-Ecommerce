# Story 1.3: Catalog API & Rendering

**Epic:** Foundation & Catalog Setup  
**User Story:** As a customer, I want to browse products and protocols on the catalog page so that I can see their prices and scientific active ingredients.

---

## Acceptance Criteria

- [ ] REST API endpoint `GET /api/catalog` exposes all products and protocols as JSON
- [ ] Catalog page (`/catalog`) renders all 5 products and 4 protocols
- [ ] Product/protocol card displays: name, price, category, INCI snippet
- [ ] Interactive modal displays full INCI nomenclature when clicked
- [ ] Tailwind styling matches premium brand (dark slate, emerald accents, glass morphism)
- [ ] Mobile-responsive (80%+ of traffic expected on mobile)
- [ ] Page loads skeleton state during API fetch
- [ ] Error handling for database failures (user-friendly message)

---

## Technical Details

### API Route: GET /api/catalog

**Request:**
```
GET /api/catalog
```

**Response (200):**
```json
{
  "status": "ok",
  "data": {
    "products": [
      {
        "id": "uuid",
        "sku": "TRIA-P1",
        "name": "Shampoo Reconstrutor",
        "price": 89.90,
        "category": "Cabelo",
        "active_ingredients": {"silk_proteins": "...", "keratin": "..."},
        "inci": "Aqua, Silicones, Keratin, Amino Acids"
      },
      ...
    ],
    "protocols": [
      {
        "id": "uuid",
        "name": "Origin",
        "price": 249.90,
        "original_price": 314.70,
        "products": ["uuid1", "uuid2", "uuid3"],
        "narrative": "Comece sua jornada..."
      },
      ...
    ]
  }
}
```

**Response (500):**
```json
{
  "status": "error",
  "message": "Failed to fetch catalog"
}
```

### Catalog Page Component (`/catalog`)

**Layout:**
1. **Header:** "Nossos Produtos & Protocolos" with navigation
2. **Products Section:**
   - Grid (3 cols desktop, 2 cols tablet, 1 col mobile)
   - Glassmorphic cards with product image placeholder, name, category badge, price
   - Click to open INCI modal
3. **Protocols Section:**
   - Grid layout same as products
   - Highlight protocol name, discount savings (original_price - price)
   - "Adicionar ao Carrinho" CTA (pending cart implementation)
4. **Skeleton Loading State:**
   - During API fetch, display placeholder cards with animate-pulse

### INCI Modal Component

**Trigger:** Click on product/protocol card  
**Content:** Full INCI nomenclature in readable format  
**Close:** Click outside modal or X button

### Styling

**Colors:**
- Background: `bg-slate-900` (dark slate)
- Cards: `glass` class (glassmorphism)
- Badges: `bg-emerald-600` (protocol names)
- Text: `text-slate-100` (light text on dark)

**Typography:**
- Headings: serif (Playfair Display)
- Body: sans-serif (Inter, Outfit)

### Files Created

- `src/app/api/catalog/route.ts` — GET /api/catalog handler
- `src/app/catalog/page.tsx` — Catalog page component
- `src/components/CatalogCard.tsx` — Product/Protocol card component
- `src/components/INIModal.tsx` — INCI nomenclature modal
- `src/app/catalog/layout.tsx` — Catalog page layout (optional)
