## Technical Details

### Stack
- **Runtime:** Node.js 20.x
- **Framework:** Next.js 14.2.3 (App Router)
- **Language:** TypeScript 5.4.x
- **Styling:** Tailwind CSS 3.4.x + PostCSS
- **Database:** Supabase PostgreSQL 15.x (schema in migrations/)
- **Testing:** Jest 29.x + React Testing Library 14.x
- **Auth:** Supabase Auth (configured, integration pending)
- **Payments:** Stripe / Pagar.me (integration pending)

### Files Created
- `package.json` — Dependencies for Next.js, Tailwind, Supabase, testing
- `tsconfig.json` — Strict TypeScript mode, path aliases (@/*)
- `tailwind.config.ts` — Premium dark palette (slate, emerald), typography
- `next.config.js` — App Router, environment variables
- `.eslintrc.json` — ESLint config extending next/core-web-vitals
- `.prettierrc` — Prettier formatting (semi, trailing comma, single quotes)
- `.lintstagedrc.json` — Lint-staged hooks for staged files
- `jest.config.js` + `jest.setup.js` — Jest + RTL setup
- `src/app/layout.tsx` — Root layout with metadata, dark mode body
- `src/app/page.tsx` — Landing page with quiz/catalog CTAs
- `src/app/globals.css` — Global styles, glass morphism, button utilities
- `src/app/api/health/route.ts` — Health check endpoint
- `src/types/index.ts` — Shared TypeScript interfaces (Product, Protocol, User, etc.)
- `src/utils/logger.ts` — Error/info/debug logging (no PII)
- `src/services/db-service.ts` — DatabaseService for Supabase operations
- `src/services/quiz-evaluator.ts` — QuizEvaluator recommendation logic
- `supabase/migrations/001_initial_schema.sql` — Full DDL + RLS policies + seed data

### Decisions Made

1. **Single App Monorepo** — Next.js handles both frontend pages and API routes under `/api/`
2. **Server-First Components** — Default to Next.js Server Components (SSR for SEO); client state only where needed
3. **Supabase for Auth & DB** — Reduces infrastructure overhead, built-in Row-Level Security
4. **Premium Color Palette** — Dark slate greys, emerald green accents matching clinical positioning
5. **Strict TypeScript** — `strict: true` enforces type safety across entire codebase
6. **No Inline SQL** — All DB operations via `DatabaseService` encapsulation layer
7. **Logger Without PII** — JSON logging for production, no diagnostic answers/names logged
