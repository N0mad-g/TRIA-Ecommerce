## Story

**As a** dev,
**I want** o projeto Next.js (App Router) inicializado com Tailwind, Framer Motion, lucide-react, deploy automático na Vercel e um gate de teste no CI,
**so that** as próximas stories tenham uma base funcional e protegida por testes antes de ir pra produção.

## Acceptance Criteria

1. Projeto Next.js (App Router, versão estável mais recente no scaffold) roda localmente (`npm run dev`) sem erros.
2. Tailwind CSS, Framer Motion e lucide-react instalados e configurados.
3. Deploy automático na Vercel a partir do branch main funcional (preview URL acessível).
4. Página inicial temporária ("health check") renderiza sem erro em produção.
5. Jest + React Testing Library + jest-axe instalados e configurados, com um teste de exemplo passando (`npm test`).
6. `README.md` documenta setup local (pré-requisitos, instalação, comandos de dev/test/build) e `.env.example` lista **os nomes** de todas as variáveis de ambiente necessárias sem valores reais.
7. Workflow `.github/workflows/test.yaml` criado, rodando `npm test`/`npm run build` em Pull Requests contra `main`.

### Environment Variables (para `.env.example`, Task 6)

[Source: architecture.md Seção 11.2 — Required Environment Variables]

```bash
# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=          # server-only, nunca exposto ao client
SUPABASE_DB_URL=                    # server-only, usado só pelo vercel-build (Story 1.2+)

# Stripe
STRIPE_SECRET_KEY=                   # server-only
STRIPE_WEBHOOK_SECRET=               # server-only
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=  # se necessário no client

# App
APP_URL=http://localhost:3000
CRON_SECRET=                         # server-only, valida chamadas do Vercel Cron (Story 1.7)
```

**Regra crítica (Seção 15.1 Coding Standards):** nenhuma dessas variáveis server-only pode ganhar o prefixo `NEXT_PUBLIC_` — é a barreira técnica do Next.js contra vazamento de secret pro bundle do client.

### Technical Constraints

- **Versão do Next.js não travada:** documentar em qualquer lugar (README, comentário) como "versão estável mais recente", nunca hardcodar um número [Source: architecture.md#1.1].
- **Sem Supabase/Stripe reais ainda:** esta story não configura contas nem chaves reais — isso é User Responsibility (PRD Seção 2.4), pré-requisito de stories futuras (1.2, 2.1), não desta.
- **`vercel-build` fica para depois:** ver nota de sequenciamento na Task 3 — não implementar o script condicional com `supabase db push` nesta story.

### Testing

[Source: architecture.md Seção 14 — Testing Strategy]

- **Localização de testes:** co-localizados com o código (`ComponentName.test.tsx` ao lado de `ComponentName.tsx`), padrão Jest + RTL.
- **Framework:** Jest + React Testing Library para componentes; `jest-axe` para violações de acessibilidade (`expect(await axe(container)).toHaveNoViolations()`).
- **Requisito desta story:** pelo menos 1 teste de exemplo passando, validando que a suíte de testes está funcional (`npm test` sem erro) — não precisa cobrir lógica de negócio ainda, já que a página de health-check é temporária.
- **E2E:** fora de escopo do MVP (decisão explícita do PRD) — não configurar nesta story.

### Completion Notes List

- **Tailwind é v4** (instalado pelo `create-next-app`) — config é **CSS-first** via `@theme` em `app/globals.css`, não `tailwind.config.ts`. Esse arquivo não existe nessa versão; a Task 2/Architecture citam o nome antigo — usar `app/globals.css` como fonte de verdade daqui pra frente.
- **Next.js 16 tem breaking changes vs. treino do modelo** (aviso do próprio scaffold, `AGENTS.md`/`CLAUDE.md`) — lido `node_modules/next/dist/docs/` antes de escrever código. Achado real: `searchParams` agora é `Promise` em Server Components (precisa `await`) — o exemplo de código da Story 1.5/Architecture Seção 8.3 estava desatualizado pra essa versão; **corrigido em `docs/architecture.md`** antes da Story 1.5 ser implementada.
- **`npm test` inicialmente pegou os testes internos do framework AIOX** (`.aiox-core/`), travando o worker (2 suites falhando, `process.exit` conflitando). Corrigido com `testPathIgnorePatterns` excluindo `.aiox-core/` no `jest.config.js`.
- **`npm audit`: 2 vulnerabilidades moderate em `postcss` transitivo do Next** — fix sugerido rebaixaria o Next pra v9 (quebra o projeto). Risco real baixo (XSS em CSS não-confiável, não é nosso caso de uso) — aceito sem ação, não é dívida técnica registrada (é decisão consciente, não esquecimento).
- **Task 3 parcialmente bloqueada:** criei `.github/workflows/test.yaml` (código, dentro do meu escopo), mas "conectar repositório à Vercel" exige a conta Vercel do usuário — não posso fazer via CLI sem credenciais/OAuth interativo. AC3 (deploy automático funcional, preview URL acessível) não pode ser verificado até o usuário conectar o repo. Status mantido `InProgress`, não `Ready for Review`, até esse bloqueio ser resolvido.
- **CodeRabbit indisponível neste ambiente** (sem binário nativo instalado, WSL com erro de systemd não relacionado ao projeto) — seguido com os `quality_gate_tools` alternativos já previstos na story (manual review + `npm run build` + `npm test`, ambos executados e passando).
- **Package.json `name`:** corrigido de `tria-scaffold-tmp` (herdado do diretório temporário usado pra contornar a restrição do `create-next-app` de não aceitar nome com maiúscula, já que o diretório do projeto é `TRIA`) para `tria`.
