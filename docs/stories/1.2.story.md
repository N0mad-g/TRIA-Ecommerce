**Epic:** Foundation & Catalog Setup  
**User Story:** As a developer, I want to define the PostgreSQL schema for users, products, protocols, and user_diagnoses so that data is persistent.

---

## Technical Details

### Database Schema Entities

**products** (5 records)
- id, sku, name, price, category, active_ingredients (JSONB), inci
- Categories: Cabelo, Styling, Barba
- All INCI nomenclature included

**protocols** (4 records)
- id, name (Origin, Reconstruct, Define, Complete)
- price, original_price, products (UUID array), narrative

**profiles**
- Links to Supabase auth.users
- Tracks last_quiz_recommendation

**user_diagnoses**
- Stores quiz answers (JSONB) and recommended protocol
- user_id nullable (allows guest diagnoses)

**subscriptions**
- Tracks active recurring billing
- Includes gateway_subscription_id for payment provider

### RLS Policies

All authentication-required tables have row-level security enabled:
- Users can only read/update own profile
- Users can only read/insert own diagnoses
- Users can only read own subscriptions

### Files Created

- `supabase/migrations/001_initial_schema.sql` — Complete DDL + RLS policies + seed data
