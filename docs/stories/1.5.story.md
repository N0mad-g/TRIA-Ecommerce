## Story

**As a** visitante,
**I want** navegar pelos produtos numa tela única, trocando de produto via setas,
**so that** eu veja detalhes e prova social sem uma página de grid separada.

## Acceptance Criteria

1. `/produtos` exibe o produto atual com nome, imagem, descrição, ativos, preço avulso.
2. Setas laterais navegam para o produto anterior/próximo do catálogo (não trocam foto do mesmo produto).
3. Bloco de métricas fixo (nota + clientes + % resultado) exibido no formato do brief.
4. Link cruzado "ou assine o Protocolo X e pague R$Y/mês" direciona para `/assinatura`.
5. Filtro de categoria (Todos, Cabelo, Barba) funcional.
6. Nenhum CTA de compra funcional ainda (ação fica para o Epic 2) — botão visível porém desabilitado/placeholder.

### Routing — Deep-linking (crítico, AC 1/2)

[Source: architecture.md Seção 8.3]

```typescript
// app/produtos/page.tsx
export default async function ProdutosPage({ searchParams }: { searchParams: { item?: string } }) {
  const products = await getProducts();
  const initialIndex = searchParams.item
    ? Math.max(0, products.findIndex((p) => p.slug === searchParams.item))
    : 0;
  return <ProductCarousel products={products} initialIndex={initialIndex} />;
}
```

### Accessibility (WCAG AA)

[Source: architecture.md Seção 8.5]

- Setas de navegação: `<button aria-label="Produto anterior/próximo">`.
- Produto atual em região `aria-live="polite"` — nome muda ao navegar, screen reader precisa perceber sem navegação de página.

### Technical Constraints

- **Sem rota dinâmica `/produtos/[slug]`** — NFR6 do PRD, navegação sempre por índice interno [Source: architecture.md Seção 8.3].
- **CTA de compra é placeholder** — não implementar `/api/checkout/one-time` nesta story (Story 2.3).
